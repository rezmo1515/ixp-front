## 🚀 نحوه استفاده

### 1. **ورود به سیستم**

```
درست برای صفحه: /login
→ انتخاب تب "ورود"
→ شماره موبایل و رمز عبور وارد کنید
→ یا استفاده از OTP
```

### 2. **ثبت نام کاربر جدید**

```
درست برای صفحه: /login
→ انتخاب تب "ثبت نام"
→ تمام فیلدها را پر کنید
→ ثبت نام
```

### 3. **استفاده از منو**

- **تست Ping:** `/ping` - تست اتصال به سرور‌ها
- **بررسی DNS:** `/dns` - بررسی رکوردهای DNS
- **WHOIS:** `/whois` - جستجوی اطلاعات دامنه
- **پیام‌ها:** `/messages` - ارسال و مدیریت پیام‌ها
- **نقش‌ها:** `/roles` - مدیریت نقش‌ها و مجوزها (فقط مدیران)

### 4. **مدیریت نقش‌ها**

```
1. برو به `/roles`
2. انتخاب یک نقش برای ویرایش
3. اضافه/حذف مجوزها
4. ذخیره تغییرات
5. یا ایجاد نقش جدید
```

### 5. **ارسال پیام**

```
1. برو به `/messages`
2. انتخاب کانال (SMS، Email، WhatsApp، etc)
3. انتخاب "ارسال به تمام" یا "کاربران خاص"
4. نوشتن پیام
5. (اختیاری) زمان‌بندی برای بعد
6. ارسال
```

## 🔐 سیستم مجوزها

### مجوزهای سیستم:

| مجوز | توضیح |
|-----|--------|
| `ping.run` | اجرای تست Ping |
| `dns.check` | بررسی DNS |
| `whois.lookup` | جستجوی WHOIS |
| `messages.send` | ارسال پیام |
| `messages.broadcast` | ارسال پخش‌شده |
| `roles.manage` | مدیریت نقش‌ها |
| `users.manage` | مدیریت کاربران |
| `logs.view` | مشاهده گزارش‌ها |

### چگونه مجوزها کار می‌کنند:

```typescript
// 1. در useAuth hook
const { hasPermission } = useAuth()

// 2. در صفحات
if (!hasPermission('roles.manage')) {
  return <AccessDenied />
}

// 3. برای دکمه‌ها
{hasPermission('messages.send') && (
  <button>ارسال پیام</button>
)}

// 4. برای منو
const nav = allNav.filter(n => 
  !n.permission || hasPermission(n.permission)
)
```

## 💡 نکات مهم

### 1. **معماری آماده برای ماژول‌های جدید**

اگر می‌خواهید ماژول جدید بسازید:
1. صفحه جدید در `src/pages/` ایجاد کنید
2. API service در `src/api/services.ts` اضافه کنید
3. Route در `src/App.tsx` اضافه کنید
4. Menu item در `src/components/layout/Layout.tsx` اضافه کنید
5. Breadcrumb item در `src/components/layout/Breadcrumb.tsx` اضافه کنید

**مستندات کامل:** دیکھیں `ARCHITECTURE.md`

### 2. **مجوزها مهم‌اند**

- **هر صفحه** باید مجوز را بررسی کند
- **هر دکمه** باید مجوز کاربر را چک کند
- **Backend** نیز باید مجوزها را بررسی کند

### 3. **RTL Support**

تمام فایل‌ها برای فارسی بهینه شده‌اند:
- `dir="rtl"` در Layout
- `dir="ltr"` برای فیلدهای شماره‌ای
- متن‌های فارسی در تمام جای

### 4. **Error Handling**

```typescript
try {
  await apiCall()
} catch (error) {
  toast.error(error.response?.data?.message || 'خطایی رخ داد')
}
```
